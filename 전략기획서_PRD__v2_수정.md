## 1. 서비스 개발 배경 및 내용

### 1-1. 개발 배경

직장인 개인투자자는 자기 포트폴리오가 지금 위험한지 아닌지를 판단할 **시간도, 도구도, 기준도** 없습니다. 하루 약 18분의 가용 시간으로는 쏟아지는 뉴스·공시를 다 읽을 수 없고, 읽어도 "내 계좌에 얼마나 영향인지"로 번역해주는 서비스가 없습니다. 시장에는 판단을 대신해주는 서비스(로보어드바이저)와 정보를 나열만 하는 서비스(뉴스·커뮤니티)만 있을 뿐, **"판단 재료를 내 포트폴리오 기준으로 가공해주는" 서비스가 없다**는 공백이 출발점입니다.

### 1-2. 서비스 내용

**한 줄 정의**: 보유 종목 관련 뉴스(ESG 포함 전체 카테고리)를 AI가 매일 읽어, 그날의 뉴스로 인해 향후 20거래일 누적 주가가 상승할지 하락할지 여부(이진 분류)와 확신도를 예측하고, 사용자의 위험 성향에 맞춰 날씨(☀️⛅🌧️⚡)로 압축해 보여주는 개인화 리스크 대시보드.

**기술 문제 정의 (재확정)**

| 항목 | 정의 |
| --- | --- |
| **문제 유형** | 지도학습 이진 분류 (회귀 아님) |
| **출력 y** | P(20거래일 누적 주가 상승) ∈ [0, 1] — 내부 계산값(지도학습 분류 스코어). 사용자에게는 **방향(상승/하락) + 확신도(강/중/약)**만 노출, 확률 수치·특정 등락폭은 노출하지 않음 |
| **입력 X — 주 신호** | 뉴스의 **카테고리**(ESG/실적·재무/산업·사업동향/문화·마케팅/기타)별로 **방향(긍정/부정)**과 **강도(severity)**를 업종별 **Materiality 여부(1/0)**에 따라 **material 신호 / immaterial 신호 두 갈래로 분리 집계**한 값 |
| **입력 X — 보조 신호** | 자본이벤트(유상증자·전환사채 등)·상장폐지 관련(감사의견 거절 등) 공시 발생 플래그 — DART `주요사항보고서` 유형 코드로 규칙 기반 태깅, 별도 ML 불필요 |
| **입력 X — 정형 피처** | 가격/거래량 피처(수익률·변동성·거래량 z-score), 거시 변수(금리·환율) |
| **모델** | DL: ① 뉴스 카테고리 판정 — zero-shot(학습 없음) / ② 관련성+방향 — **Inference API / snunlp/KR-FinBert-SC 감성 분류기 활용** / ML: XGBoost (지도학습 이진 분류, 실제 학습) |
| **평가지표** | PR-AUC, Accuracy, AUC-ROC, Brier Score (20거래일 누적 방향 예측에서 리스크 피처 추가 시 경보 신호의 실질적인 정밀도와 재현성 궤적을 엄격히 평가하기 위해 PR-AUC를 핵심 평가지표로 채택) |
| **베이스라인** | ① 양성 비율(prior, 상승 기준율) ② 가격 피처만(뉴스·자본이벤트 제외) |
| **검증** | Ablation 3단 비교: 베이스라인 → +뉴스카테고리 신호 → +뉴스카테고리+자본이벤트/상장폐지 |

**왜 자본이벤트·상장폐지 신호를 함께 쓰는가**: 도메인이해서에서 지적했듯 ESG(특히 환경·사회)는 개별적으로는 주가 파급력이 약할 수 있습니다. 그런데 뉴스·공시를 수집하는 과정에서 자본이벤트·상장폐지 관련 공시는 **추가 비용 없이 같이 확보되는 데이터**이므로, 이를 보조 피처로 병행하면 ESG 신호가 약할 때의 리스크를 보완할 수 있고, Ablation 비교도 "ESG만" vs "ESG+보조신호" 2단으로 더 설득력 있게 구성됩니다.

**업종별 뉴스 영향도 차등 반영 (Khan, Serafeim & Yoon, 2016 방법론을 전체 카테고리로 응용·확장)**: 참고 논문은 SASB의 업종별 Materiality Map을 이용해 "ESG 이슈 카테고리 × 업종" 조합마다 그 이슈가 해당 업종에서 재무적으로 중요한지(material)를 **1(중요) / 0(비중요)**로 이진 분류하고, material로 분류된 이슈들의 성과만 모아 `Materiality Score`를, 나머지 이슈들의 성과를 모아 `Immateriality Score`를 **별도로 산출**합니다(연속 계수를 곱하는 가중합 방식이 아니라, 두 트랙으로 분리 집계하는 방식). 논문은 material 지표만 미래 주가와 유의한 관계를 보인다는 것을 실증했습니다.

이 서비스는 **이 이진 분류·분리 집계 아이디어를 ESG 한정이 아니라 뉴스 전체 카테고리로 확장 적용**합니다. (⚠️ 논문 원안은 ESG 이슈에 한정된 방법론이므로, "실적/재무"나 "문화/마케팅"까지 확장하는 것은 논문을 그대로 재현하는 게 아니라 그 아이디어를 응용한 것임을 명확히 구분합니다.)

- 뉴스는 **사용자에게도 노출되는 카테고리**로 분류됩니다: `ESG`(환경·사회·지배구조) / `실적·재무`(실적발표, 매출·이익, 자본이벤트) / `산업·사업동향`(신사업, 제품, 파트너십, 수주) / `문화·마케팅`(브랜드, 광고, 스폰서십) / `기타`. 종목 상세 화면에 `[ESG]`, `[문화]`처럼 카테고리 태그가 그대로 표시됩니다(과거 결정사항 #3의 "세부 카테고리 비노출" 원칙을 대체).
- 업종별 **Materiality Map(정적 lookup table, "업종 × 뉴스 카테고리 → 1/0")**을 참조해 해당 사건이 그 업종에서 material인지 판정합니다.
- material(=1)로 판정된 사건은 `category_material_value`(방향×강도의 누적)로, immaterial(=0)로 판정된 사건은 `category_immaterial_value`로 **분리 집계**하여 최종 피처 두 개를 산출합니다. 두 피처 모두 XGBoost에 입력하고, 어떤 피처가 실제로 예측력에 기여하는지는 모델이 학습 과정에서 판단하게 합니다.

**절대 원칙 (Non-Goal, 변경 없음)**: 종목 추천 안 함, 매수/매도 시점 제시 안 함, 목표가·수익률 보장 안 함, 자동 매매 안 함, 시장 전체 방향 예측 안 함.

### 2-2. 정보 아키텍처 — 이원화된 진단 (변경 없음)

체질(월 단위, 성향 vs 포트폴리오 구조 적합도)과 날씨(매일, 뉴스 기반 다음 거래일 방향+확신도)는 화면을 분리 유지합니다.

### 2-3. 종목 상세 화면 — 근거 표시 방식 개정

Direction Engine 카드 체계를 폐기했으므로, 종목 상세는 **날씨 + 방향/확신도 + 근거 한 줄(카테고리 태그 포함)**로 통일합니다.

```
삼성전자  ⚡  하락 위험 신호
─────────────────────────────
다음 거래일 전망      하락  ·  확신도 강

감지된 근거
· [ESG·부정] xx기업 파업 관련 보도 — 07/20  [원문 →]
  └ 반도체 업종의 핵심(material) 이슈로 판정되어 가중 반영
· [문화·긍정] xx기업 OOO 소식 — 07/20  [원문 →]
· [공시·보조신호] 유상증자 결정(자본이벤트) — 07/20  [원문 →]
· [실적·재무·부정] 업황 둔화 우려 — 07/18       [원문 →]

참고: 이 신호가 뜬 과거 사례의 적중률은
      최근 6개월 기준 58%였습니다.
```

- 근거는 **"[카테고리·방향] 제목"** 형식으로 표시합니다. 카테고리 태그(`ESG`/`실적·재무`/`산업·사업동향`/`문화·마케팅`/`기타`)는 사용자에게 그대로 노출합니다(과거 결정사항 #3을 대체 — 세부 pillar 비노출 원칙은 더 이상 적용하지 않음).
- 노출하지 않는 것은 **확률 수치(%)와 특정 기간·특정 등락폭**입니다. "다음 거래일 방향 + 확신도(강/중/약)"까지만 보여주고, 내부 계산에 쓰인 P(상승) 원값은 화면에 표시하지 않습니다.
- 자본이벤트/상장폐지 관련 공시는 "보조신호"로 명시적으로 라벨링해, 뉴스 카테고리 신호와 출처가 다름을 사용자가 구분할 수 있게 합니다.

### 2-4. UX 원칙 (변경 없음)
결론 먼저·비중 순 정렬·모든 판정에 근거 한 줄·확률 원값 병기·면책 문구 상시 고정·빈 상태도 콘텐츠로 표현.

## 3. 비즈니스 분석

### 3-1. 시장 기회

국내 활동계좌는 2026년 상반기 1억 개를 돌파했고, 반기 만에 1,000만 개 이상 늘었습니다. 로보어드바이저 시장도 커지고 있지만 전부 일임형이라 "직접 고른 종목 진단"은 공백입니다. 여기에 더해, **ESG 평가 시장 자체도 확인이 필요합니다.**

국내 ESG 평가는 한국ESG기준원(KCGS)·서스틴베스트·한국ESG연구소 등이 담당하는데, 이들은 전부 **기관투자자·기업 대상 B2B 리서치·자문 서비스**이며 개인투자자가 자기 보유 종목의 ESG 리스크를 실시간으로 확인할 채널은 사실상 없습니다. 즉 "ESG 평가는 있지만 개인용은 없다"는 두 번째 공백이 확인되며, 우리 서비스는 이 지점도 함께 겨냥합니다.

### 3-2. 수익모델 가설 (범위 밖, 방향성만)

프리미엄 구독(기본 날씨 무료, 종목 상세·ESG 히스토리 유료), 증권사 제휴/화이트라벨, B2B ESG 리스크 API 라이선싱 — 이번 스프린트에서는 검증하지 않습니다.

### 3-3. 경쟁사 분석 (개정)

| 구분 | 대표 서비스 | 하는 일 | 우리와 다른 점 |
| --- | --- | --- | --- |
| **일임형 로보어드바이저** | 핀트, 파운트, 에임, 콴텍 | 성향 진단 후 자산을 대신 운용 | 돈을 맡기지 않음. 보유 종목만 진단 |

### 4-2. AI 파이프라인 (개정)

```
[배치, 매일 07:00]
DART 공시 + 뉴스 텍스트
        │
        ├─ ① 규칙 기반 태깅: DART 유형코드 → CAPITAL_EVENT / DELISTING_RELATED (ML 불필요)
        │
        └─ ② 뉴스 텍스트 분류 (DL, 3단계 — 카테고리 판정을 가장 먼저 수행)
               2-1) **카테고리 판정 (가장 먼저, HuggingFace zero-shot, 학습 없음)**
                    → `ESG` / `실적·재무` / `산업·사업동향` / `문화·마케팅` / `기타` 중 최적 라벨 산출
                    → 이 결과가 사용자 화면에 그대로 노출되는 태그이자, 이후 처리 우선순위/Materiality 조회의 키가 됨
               2-2) 관련성(뉴스 관련/비관련) + 방향(긍정/부정) 판정 (**linear probing, 실제 학습**)
                    → 사전학습 임베딩(KLUE-RoBERTa 등, 가중치 고정) 위에 경량 분류기 head만 별도 라벨링 데이터로 학습
                    → 분류기 confidence를 강도(severity, 0~1)로 사용
                    → (시간 부족 시 폴백) ESG·실적/재무 카테고리부터 우선 처리, 나머지는 zero-shot으로 대체
                        │
                        ▼ 업종별 Materiality Map 조회 (업종 × 뉴스 카테고리 → 1/0, 정적 lookup table)
                        │
                        ├─ material(1) → category_material_value 누적
                        └─ immaterial(0) → category_immaterial_value 누적
                        │
                 종목별 일별 뉴스 피처(material/immaterial 분리, 카테고리별) + 보조신호 플래그
                        │
가격·거래량 + 거시지표 ──┐         │
                        ▼         ▼
                    XGBoost 추론 (ML)
                        │
                        ▼
         MongoDB: daily_risk_score (다음 거래일 방향 + 확신도)
                        │
[요청 시]
포트폴리오 로드 → 종목별 방향/확신도 조회 → 성향 임계치 적용 → 날씨 산출
                       + event_study_stats 조회(적중률) → LLM 브리핑(설명만)

[별도 배치, 주 1회]
과거 (업종, news_category, direction, is_material) 조합별 데이터 집계
  → 다음날 실제 방향 일치율(적중률) 계산 → MongoDB: event_study_stats
```

**라벨(정답) 정의**: 특정 종목의 `label_direction_next_day`(지도학습 분류 타깃)는 "향후 20거래일 동안의 누적 수익률이 0보다 큰지(상승=1), 아닌지(하락/보합=0)"로 정의합니다. 시계열 학습의 미래 정보 누수를 차단하기 위해 검증 시 엠바고(gap=20)를 엄격히 적용합니다.

**Direction Engine의 위치 & 이벤트 스터디의 승격**
- Direction Engine(`REVIEW/HOLD/WATCH/REDUCE_REVIEW`)은 **파이프라인에서 완전히 제거**되었습니다. 최종 산출은 방향(up/down)과 확신도입니다.
- 이벤트 스터디(AR/CAR)는 v2.0에서는 "1회성 오프라인 EDA"였지만, **정기 배치(주 1회 권장)로 승격**합니다. `(업종, news_category, direction, is_material)` 조합별로 과거 데이터를 모아 "이 유형의 사건 다음날 실제로 같은 방향으로 움직인 비율(적중률)"을 계산해 `event_study_stats` 컬렉션에 저장합니다. 종목 상세 화면의 "과거 사례 적중률 58%" 같은 근거 문구는 이 계산 결과를 그대로 조회한 값입니다. (일별 배치처럼 매일 다시 계산할 필요는 없고, 표본이 쌓이는 속도에 맞춰 주 1회 정도로 충분합니다.)

**바이브코딩 난이도 기준 적용**
- 뉴스 텍스트 분류는 **세 단계로 나눠서, 카테고리 판정을 가장 먼저** 처리합니다.
  - **카테고리 판정 (1순위)**: HuggingFace Inference API의 zero-shot-classification 태스크(`joeddav/xlm-roberta-large-xnli` 등)로 처리, 학습 없음. candidate_labels = `["ESG 관련", "실적/재무 관련", "산업/사업동향 관련", "문화/마케팅 관련", "기타"]`. 이 결과가 사용자 화면 태그이자 이후 처리·Materiality 조회의 라우팅 키이므로 가장 먼저 수행합니다.
  - **관련성 + 방향(긍정/부정) 판정**: **linear probing**으로 처리합니다. 사전학습 모델(KLUE-RoBERTa 등)의 **가중치는 전부 고정(freeze)**한 채 문장 임베딩만 추출하고, 그 위에 얹는 **작은 분류기 head(로지스틱회귀 또는 1~2층 MLP)만 팀이 직접 라벨링한 소량 데이터(수백 건)로 학습**시킵니다. 분류기의 출력 확률(softmax confidence)을 그대로 강도(severity, 0~1)로 사용합니다.
  - **처리 우선순위(시간 부족 시 폴백)**: `ESG`, `실적·재무` 카테고리는 linear probing까지 정교하게 처리하고, `산업·사업동향`/`문화·마케팅`/`기타`는 시간이 부족하면 카테고리 판정까지만 하고 방향 판정은 zero-shot(`["긍정적인 사건", "부정적인 사건"]`)으로 대체합니다.
- 자본이벤트/상장폐지 태깅은 ML이 아니라 **DART 공시 유형 코드 매핑 규칙**(if-else 수준)으로 처리해, 비전공자도 로직을 바로 읽을 수 있게 합니다.
- Materiality Map은 "업종 × 뉴스 카테고리(5종)" 매트릭스에 **1(material)/0(immaterial)**만 채워 넣는 **정적 lookup table**(JSON/CSV)로 관리하며, 학습되는 파라미터가 아닙니다. `ESG`, `실적·재무` 칸을 먼저 채우고, 나머지 칸은 시간이 부족하면 전부 `0`(immaterial)으로 두는 것을 기본 폴백으로 합니다.
- 실시간 뉴스 수집은 **NAVER API HUB**(`naverapihub.apigw.ntruss.com/search/v1/news`)를 사용합니다. 옛 `developers.naver.com` 개별 신청 방식은 NAVER Cloud Platform 통합 플랫폼으로 이관되었으므로, 새 콘솔에서 키를 발급받아야 합니다.

### 4-3. 데이터 스키마 (개정)

① 뉴스 사건 피처 (DL 출력, DB 저장 대상 — 카테고리 태그는 사용자 노출용)
```json
{
  "ticker": "005930",
  "date": "2026-07-20",
  "news_related": true,
  "news_direction": "negative",
  "news_severity": 0.72,
  "news_category": "ESG",
  "is_material": 1,
  "confidence_score": 0.74,
  "evidence": [
    { "title": "string", "source": "string", "url": "string", "published_at": "datetime", "type": "NEWS" }
  ]
}
```
- `news_direction`/`news_severity`: linear probing 분류기(사전학습 임베딩 고정 + 학습된 경량 head)의 출력. `news_severity`는 해당 클래스의 softmax confidence(0~1)를 그대로 사용.
- `news_category`: `ESG` / `실적·재무` / `산업·사업동향` / `문화·마케팅` / `기타` 중 하나 — HuggingFace zero-shot(학습 없음)으로 가장 먼저 산출하며, **사용자 화면에 `[ESG]`, `[문화·마케팅]`처럼 그대로 노출됩니다.**
- `is_material`: Materiality Map(`{업종, news_category} → 1/0`)을 조회한 결과. **1 = 이 업종에서 재무적으로 중요한 카테고리, 0 = 중요하지 않은 카테고리** (Khan, Serafeim & Yoon, 2016 아이디어를 전체 카테고리로 확장 적용).
- 종목별 일별 집계 시 `is_material` 값에 따라 아래처럼 두 피처로 분리 합산:
  - `category_material_value = Σ(news_direction × news_severity)` (해당 일 `is_material == 1`인 사건들만)
  - `category_immaterial_value = Σ(news_direction × news_severity)` (해당 일 `is_material == 0`인 사건들만)

② 보조 신호(자본이벤트/상장폐지, 규칙 기반)
```json
{
  "ticker": "005930",
  "date": "2026-07-20",
  "event_type": "CAPITAL_EVENT",
  "disclosure_category": "유상증자 결정",
  "detected_by": "RULE_BASED_DART_TYPE_MAP",
  "evidence": { "title": "string", "url": "string", "published_at": "datetime", "type": "SUPPLEMENTARY" }
}
```

③ 가격·거시 피처 (변경 없음)
```json
{
  "ticker": "005930", "date": "2026-07-20",
  "log_return_1d": 0.012, "volatility_20d": 0.284,
  "volume_zscore": 2.1, "beta_60d": 1.15,
  "sector": "반도체", "macro_rate": 3.5, "macro_fx": 1385.2
}
```

④ ML 학습 데이터 행
```json
{
  "ticker": "005930", "date": "2026-07-20",
  "features": {
    "category_material_value": 0.61,
    "category_immaterial_value": -0.08,
    "capital_event_flag": 1,
    "delisting_related_flag": 0,
    "...가격/거시 피처...": "..."
  },
  "label_direction_next_day": 1
}
```

⑤ 모델 출력 (`daily_risk_score`)
```json
{
  "ticker": "005930", "date": "2026-07-20",
  "prob_up": 0.24,
  "direction": "down",
  "confidence_tier": "strong",
  "top_evidence": ["...①②의 evidence 참조..."],
  "model_version": "xgb_v3_news_category_ablation"
}
```
- `prob_up`: 내부 계산값(20거래일 누적 주가 상승 확률). **DB에는 저장하지만 사용자 화면에는 노출하지 않습니다.**
- `direction`: `up`/`down` — `prob_up`이 0.5보다 크면 `up` (상승), 작으면 `down` (하락).
- `confidence_tier`: `prob_up`이 0.5에서 얼마나 떨어져 있는지로 산출하는 등급. 예) `|prob_up - 0.5| < 0.1` → `weak`, `0.1~0.25` → `medium`, `>0.25` → `strong`. 화면에는 강/중/약으로 표시.

⑥ 이벤트 스터디 통계 (`event_study_stats`, 주 1회 배치로 갱신)
```json
{
  "industry": "반도체",
  "news_category": "ESG",
  "direction": "negative",
  "is_material": 1,
  "sample_size": 42,
  "hit_rate": 0.58,
  "window_months": 6,
  "updated_at": "2026-07-20"
}
```
- `hit_rate`: 이 조합(업종×카테고리×방향×material 여부)의 사건이 발생한 다음 거래일에, 실제로 같은 방향으로 움직인 비율. 종목 상세 화면의 "과거 사례 적중률" 근거로 그대로 조회해서 씁니다.
- `sample_size`가 너무 작으면(예: 5건 미만) 신뢰하기 어려우므로, 화면에는 표시하지 않고 "데이터 축적 중" 배지로 대체합니다 (4-4 Fallback 참고).

⑦~⑧ 사용자·포트폴리오·화면 응답 스키마는 v1.0과 동일 (변경 없음).

### 4-4. Fallback 전략 (개정 사항만 추가)

| 상황 | Fallback |
| --- | --- |
| 뉴스 관련 텍스트 0건 | 보조 신호(자본이벤트/상장폐지)와 가격 피처만으로 방향 산출 + "뉴스 신호 없음" 표기 |
| 뉴스 분류 신뢰도 낮음(`confidence_score` 낮음) | 해당 피처의 가중치를 낮춰 반영, "참고용 신호" 배지 |
| 업종-Materiality 매핑 누락(해당 업종×뉴스 카테고리 조합이 lookup table에 없음) | 보수적으로 `is_material = 0`(immaterial)으로 처리해 material 신호를 과대 계상하지 않음, 매핑 테이블 갱신 필요 플래그 |
| 카테고리별 Materiality Map 채우기가 시간 내에 다 안 끝남 | `ESG`, `실적·재무`부터 우선 채우고, `산업·사업동향`/`문화·마케팅`/`기타`는 전부 `0`(immaterial) 기본값으로 두고 다음 스프린트로 이월 |
| linear probing 방향 판정이 특정 카테고리(예: 문화·마케팅)에서 정확도가 낮음 | 해당 카테고리만 zero-shot(`["긍정적인 사건", "부정적인 사건"]`)으로 잠정 대체 |
| 이벤트 스터디 표본이 너무 적음(`sample_size` < 5) | "적중률" 문구 대신 "데이터 축적 중" 배지로 대체, 확정적이지 않은 수치를 보여주지 않음 |
| (기존 항목 — 변경 없음) 설문 미완료, 모델 커버리지 밖, 배치 실패 등 | v1.0과 동일 |

### 4-5. 데이터 보안 (변경 없음 + 1건 추가)
기존 원칙(원문 미저장, PII 최소화, API 키 분리, 금칙어 필터)에 더해: **ESG 관련 텍스트도 뉴스와 동일하게 원문을 저장하지 않고 방향·강도·메타데이터만 저장**합니다.

### 4-6. 어드민 기능 (개정 사항만 추가)
기존 기능(배치 모니터링, 모델 성능 대시보드, Ablation 결과 뷰, 더미 계정 관리, 금칙어 관리, 커버리지 현황)에 **Materiality Map(업종 × 뉴스 카테고리 → 1/0) 편집 기능**과 **이벤트 스터디 통계(`event_study_stats`) 조회 화면**을 추가합니다(팀이 발표 전 특정 셀 값을 0↔1로 뒤집어 조정하거나, 어떤 조합이 표본 부족 상태인지 한눈에 확인할 수 있도록 텍스트/CSV 파일 기반으로 충분).

## 5. 서비스 전략

### 5-1. 포지셔닝 (개정)
"기관투자자만 보던 ESG 리스크 정보를, 개인이 이미 보유한 종목 기준으로, 오늘의 확률과 근거로 설명한다." — 로보어드바이저(위임형)와 ESG 평가기관(B2B) 사이의 공백을 동시에 겨냥.

### 5-2. 차별화 전략 (개정)
1. **ESG + 보조신호 결합** — ESG 단독의 약한 신호를 자본이벤트·상장폐지 태깅으로 보완
2. **업종별 Materiality 차등 반영** — 모든 종목에 동일한 ESG 잣대를 적용하지 않음
3. **분류 문제 유지** — "얼마나 위험한가"만 말해 판단 대행 논란 원천 차단
4. **개인용 채널 부재 공략** — ESG 평가기관이 닿지 않는 개인투자자 시장을 정면 타겟

### 5-3. 개발 난이도 원칙 (개정)
- 뉴스 분류·감성 추론: 카테고리는 zero-shot API(학습 없음, 가장 먼저 처리), 관련성+방향은 **linear probing**(임베딩 고정 + 경량 head만 학습, ESG·실적재무 우선) — 전체 모델 파인튜닝은 없음 (Antigravity/Colab에서 무리 없는 수준)
- 자본이벤트/상장폐지 태깅: 규칙 기반 매핑, ML 불필요
- Materiality Map: 업종×뉴스 카테고리 → 1(material)/0(immaterial), 학습 파라미터가 아닌 정적 lookup table
- 예측 목표: "20거래일 누적 주가 등락(상승/하락) 방향" 이진 분류 라벨로 단순화 — 지도학습 기반의 이진 분류 문제로 정의하여 예측력 검증
- 이벤트 스터디: 정기 배치(주 1회)로 승격, `(업종×카테고리×방향×material) → 적중률` 조회 테이블만 계산해 저장 — 복잡한 통계모델 없이 단순 비율 계산 수준으로 난이도 유지
- 포트폴리오 최적화 비교(HRP/ERC/1N/블랙-리터만): 전부 제외
- **논문 근거는 유지**: 코드/설계 결정마다 어떤 논문 개념에 대응하는지 주석·문서로 남김. Materiality 판정은 Khan·Serafeim·Yoon(2016)의 **업종×이슈 material(1)/immaterial(0) 이진 분류 + material/immaterial 분리 집계** 방법론을 **ESG 한정이 아니라 뉴스 전체 카테고리로 응용·확장**해 채택함을 명시(원 논문은 ESG 이슈 한정 방법론이라는 점을 문서에 구분해서 남김). 다른 항목(이벤트 스터디, 포트폴리오 최적화 등)은 기존 원칙대로 오프라인 검증/v2 이월 등으로 단순화 유지

### 5-4. 리스크 및 대응 (개정)

| 리스크 | 대응 |
| --- | --- |
| 뉴스 신호만으로 성능 개선이 미미할 가능성 | 자본이벤트/상장폐지 보조신호를 함께 반영해 Ablation을 3단으로 설계, 어느 조합이 기여하는지 명확히 분리해 발표 |
| 뉴스 분류 정확도가 낮을 가능성(한국어 라벨 데이터 부족) | "관련/비관련 + 방향 + 강도"까지만 요구 수준을 낮춰 파인튜닝 없이도 합리적 성능 확보. 카테고리(5분류) 판정은 별도 zero-shot 프롬프트 한 단계 추가일 뿐 파인튜닝을 요구하지 않으며, 오분류되어도 "immaterial로 과소 분류"되는 정도(0으로 fallback)라 최악의 경우에도 신호가 약해질 뿐 왜곡되지는 않음 |
| ESG 한정에서 뉴스 전체 카테고리로 범위가 넓어지면서 4일 안에 다 못 채울 가능성 | `ESG`, `실적·재무` 카테고리부터 Materiality Map·라벨링을 우선 완성하고, 나머지 카테고리는 "카테고리 태그 노출까지만" 하고 material 집계·정교한 방향판정은 다음 스프린트로 이월(4-4 Fallback 참고) |
| "다음 거래일 방향"은 예측 호라이즌이 짧아 주가 움직임 자체가 노이즈(사실상 랜덤워크)에 가까울 수 있음 | 베이스라인(상승 기준율 prior)과의 비교를 KPI에 명시해, 개선폭이 작아도 "무의미한 수준인지 유의미한 수준인지"를 Accuracy/AUC-ROC로 투명하게 제시. 개선이 안 보이면 "단기 방향 예측의 한계"로 발표에서 솔직히 설명 |
| 빅카인즈 확보 실패 | 기존과 동일 — DART 공시(뉴스 관련 문구 포함) 단독 플랜B |
| linear probing용 라벨링 데이터가 부족하거나 라벨링 시간이 모자랄 가능성 | 목표를 카테고리별 수백 건 수준으로 낮추고(ESG·실적재무 우선), 부족하면 해당 카테고리는 zero-shot classification으로 잠정 대체 후 라벨이 쌓이는 대로 linear probing으로 교체 |
| 발표 시 "왜 ESG 말고 다른 카테고리까지 보냐" 질의 | Khan·Serafeim·Yoon(2016) 아이디어를 응용·확장한 것임을 명시하고, 개별 카테고리(특히 ESG)는 단독으로는 파급력이 약할 수 있어 보조신호로 보완했다는 점을 근거로 답변 준비 |

### 5-5. v2 이후 확장 로드맵 (개정)
- E/S/G 세부 pillar 분류로 세분화(ESGBERT류 파인튜닝)
- "다음 거래일 방향"을 넘어 특정 기간·특정 등락폭(예: N거래일 내 -10% 하락 확률)까지 정량 예측하는 상세 버전
- Direction Engine 부활 검토(날씨와 병행 여부 재논의)
- 이벤트 스터디 근거에 구체적 등락폭(%)까지 표시하는 상세 버전 (현재는 적중률만 노출)
- HRP/ERC/1/N/블랙-리터만 기반 포트폴리오 비교·최적화 제안
- 실제 증권사 계좌 연동(마이데이터)
